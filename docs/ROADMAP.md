# 🚀 Gabojago 서비스 개발 & 앱 출시 로드맵

## 🎯 목표

1. **PC 환경**: 펫프렌즈 스타일의 직관적인 정보 제공 및 넓은 레이아웃 활용
2. **Mobile Web**: 네이티브 앱과 동일한 UX/UI (Web App 방식)
3. **App Store**: 안드로이드/iOS 스토어 정식 출시 (Hybrid App)

## 🗓️ Phase 1: UI/UX 구조화 & 반응형 설계 (현재 단계)

**목표**: 하나의 코드(Next.js)로 PC와 모바일에서 다른 레이아웃 보여주기

- [ ] **반응형 레이아웃 시스템 구축** (`TailwindCSS` 활용)
  - PC: 중앙 정렬 컨텐츠 + 좌우 여백/배너 (펫프렌즈 스타일)
  - Mobile: 꽉 찬 화면, 하단 탭바(Bottom Navigation) 노출
- [ ] 공통 디자인 시스템 (Color, Typography) 정의
- [ ] **디바이스 감지**: 접속 환경에 따라 PC/Mobile 뷰 자동 전환 로직 구현

## 🗓️ Phase 2: 핵심 기능 개발 (MVP)

**목표**: 사용자가 실제 서비스를 이용할 수 있는 기능 완성

- [ ] **사용자 인증 (Auth)**:
  - 로그인/회원가입 (소셜 로그인 포함)
  - 백엔드(Rails)와 JWT 연동
- [ ] **메인 홈 화면**:
  - 배너 슬라이더, 추천 카테고리
  - 무한 스크롤 리스트 (상품/컨텐츠)
- [ ] **상세 페이지**:
  - 이미지 갤러리, 상세 정보, 리뷰 섹션
  - 구매하기/예약하기 플로팅 버튼(FAB)

## 🗓️ Phase 3: 네이티브 기능 연동 (App 화)

**목표**: 웹사이트를 '앱'으로 포장하기 (Capacitor 사용)

- [ ] **Capacitor 설치 및 설정**: Next.js 프로젝트를 Android/iOS 프로젝트로 래핑
- [ ] **네이티브 기능 연동**:
  - 푸시 알림 (Push Notifications)
  - 카메라/갤러리 접근
  - 소셜 공유 기능
- [ ] **오프라인 처리**: 네트워크 끊김 시 예외 처리 화면

## 🗓️ Phase 4: 스토어 심사 및 출시

**목표**: 구글 플레이스토어 & 애플 앱스토어 등록

- [ ] **앱 아이콘 & 스플래시 스크린(로딩화면)** 제작 및 적용
- [ ] **빌드 및 서명**:
  - Android: APK/AAB 파일 생성 및 서명
  - iOS: Xcode 아카이빙 및 TestFlight 업로드
- [ ] **스토어 등록 정보 작성**: 스크린샷, 설명, 개인정보처리방침
- [ ] **심사 요청 및 피드백 대응**

---

## 🛠️ 기술 전략: Next.js + Capacitor

- **PC View**: 브라우저 폭이 넓을 때(md 이상)는 펫프렌즈처럼 중앙 컨텐츠 영역(Max-width 480~600px)을 유지하고, 배경이나 사이드에 추가 정보를 배치하는 전략 사용.
- **Mobile View**: 브라우저 폭이 좁을 때(md 미만)는 100% width를 사용하고 하단 탭바(Bottom Tab)를 활성화.
